SONGSLEUTHS - DESIGN DNAProject: SongSleuths Design Document / UI & Visual System Course: CSCI 375 — Howard University — Spring 2026 Authors: Paris, Thane, Adele, Miles, Langyia Version: v1.0 — March 2026 Tech Stack: React 18, Tailwind, Django, Firebase 01 DESIGN PHILOSOPHYMinimalism: SongSleuths borrows its visual discipline from Wordle: one idea per screen, no chrome, no clutter.Focus: The UI should feel like it got out of the way. Every pixel that isn't the puzzle is a distraction.Aesthetic: The colorway is a direct nod to Spotify — near-black surfaces, Spotify green for every accent that matters, and white for the content itself.Simplicity: Two fonts, three shades of gray, one green. That's the whole system.Layout: Radical whitespace — breathing room is part of the design, not a mistake.Interaction: Green means go — #1DB954 is reserved for interactive elements, active states, and positive feedback only. No decorative elements — no gradients, shadows, borders for borders' sake.Motion: One animation per interaction — tiles reveal with a single flip, nothing more.02 COLOR SYSTEMFive colors. No others.Spotify Green (#1DB954): Primary action, correct guess reveal, active state, CTA buttons, links.Deep Black (#121212): Page background, nav bar, dark surface cards.Surface Black (#282828): Card backgrounds, tile default state, input fields.Body Gray (#535353): Secondary text, metadata, placeholder copy.Light Gray (#B3B3B3): Tertiary text, disabled states, dividers.Pure White (#FFFFFF): Primary text on dark surfaces, tile letter fills.State Colors (Puzzle Tiles):Unrevealed (#282828): Default tile — letter hidden, dark surface.Revealed (#1DB954): Correct letter guessed — fills green, letter shows white.Wrong guess (#535353): Incorrect letter — keyboard key darkens, counter decrements.Failed tile (#191414): 3 attempts used — tile stays dark, answer auto-reveals in white.03 TYPOGRAPHYDisplay Font: Courier New — all caps, tracked — used for all UI labels, navigation, headings, and button text.Body Font: Georgia — sentence case — used for descriptions, user-generated content, puzzle text, and challenge copy.Font Weights: Two weights only — Regular (400) and Bold (700). Never 500, 600, or 800.Letter Spacing: Headings: +0.08em tracked. Body: default. Tile letters: +0.05em.04 SPACING & GRIDBase unit: 8px. All spacing is a multiple of 8.Layout Constraint: Max content width: 640px centered — same constraint as Wordle.Mobile Layout: 16px horizontal margin, full-width content.Tablet+ Layout: auto horizontal margins, 640px column.Navigation: full width, 64px height, content padded to match column.05 COMPONENT SPECSPuzzle TileSize: 52px × 52px desktop / 44px × 44px mobile.Gap: 6px between.Border: 2px solid #535353 (unrevealed) / none (revealed).Radius: 4px.Text: Courier New Bold 28px, color #FFFFFF, centered.Letter KeyboardLayout: QWERTY rows, centered, same as Wordle.Key Size: 40px × 52px desktop / 32px × 44px mobile.Font: Courier New Bold 14px.Default State: Background #282828, text #FFFFFF.Primary ButtonBackground: #1DB954.Text: Courier New Bold 15px ALL CAPS, color #FFFFFF.Padding: 14px 32px.Radius: 500px (pill).Nav BarHeight: 64px.Background: #121212.Branding: SONGSLEUTHS in Courier New Bold 20px, color #1DB954.06 MOTION & ANIMATIONAnimations must earn their place. Every transition communicates something — a state change, a response, a result. Nothing animates for decoration.Tile letter reveal (correct): 400ms duration, ease-in-out, staggered per tile at 60ms.Wrong guess shake: 300ms duration, ease-out — tiles shake horizontally, no color change.Keyboard key depress: 80ms duration, ease — scale(0.95) on press, restore on release.Page route transition: 200ms duration, fade opacity 0 → 1.Score tally on results: 600ms duration, Count-up number animation.Attempt counter decrement: 200ms duration, Scale bounce on the counter badge.
